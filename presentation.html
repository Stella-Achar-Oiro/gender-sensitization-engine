02
Problem

Gender bias in African-language text

The gap: Existing NLP bias tools are built for English, on Western datasets. African languages — especially Swahili and Kikuyu — have near-zero coverage.
language

Language gap

Swahili is spoken by 200M people. No production-grade gender bias detector existed before JuaKazi.

find_replace

Correction gap

Other tools stop at detection. JuaKazi produces a corrected rewrite with a plain-language explanation.

work

Labour market context

Biased job ads and HR text in East Africa reinforce occupational segregation. JuaKazi targets this domain.

hub

Integration layer

AIBRIDGE plan: wrap other teams' detectors with JuaKazi's correction layer. One API for detection + fix.

Bias types we address

report Derogation

"Mwanamke ni nyumba." — women belong in the home. Explicit, highest priority.

warning Occupational stereotype

"Daktari wa kiume" (male doctor) — implies doctoring is a male role.

info Generic masculine

"Each employee must submit his report." — masculine default erases women.

shield Counter-stereotype

"Mwanamke mhandisi alitengeneza daraja." — preserved as positive signal, never corrected.

Key design principles
  • check_circleDetection → Correction → Explanation in one API call
  • check_circleContextChecker gates all rules — 10 conditions prevent FPs in quotes, historical text, biographical references, etc.
  • check_circleseverity=warn never sets has_bias_detected=True
  • check_circleCounter-stereotype patterns preserved, never corrected
  • check_circleML Stage 2 is warn-only fallback — rules always take precedence
  • check_circleEvery rewrite audit-logged to audit_logs/rewrites.jsonl
  • check_circleAIBRIDGE-compliant 24-column schema on all ground truth files
Input text
  → BiasDetector.detect_bias()
      ├── DEROGATION_PATTERNS    # highest priority
      ├── COUNTER_STEREOTYPE      # preserve, no correction
      └── Lexicon rules           # ContextChecker gated
            ├── severity=replace → has_bias_detected=True
            └── severity=warn   → informational only

  Stage 2 (SW only, JUAKAZI_ML_MODEL set):
  → MLBiasDetector.classify()
      └── threshold=0.75 → warn_edits only

edit_note Endpoints
POST/rewrite
GET/ — Gradio demo UI
Request body
{
  "id": "req-001",
  "lang": "sw",
  "text": "Daktari wa kiume alipima mgonjwa.",
  "region_dialect": "kenya"
}
Response
{
  "rewrite": "Daktari alipima mgonjwa.",
  "has_bias_detected": true,
  "confidence": 0.85,
  "source": "rules",
  "reason": "Occupational gender marking removed",
  "edits": [{
    "from": "Daktari wa kiume",
    "to": "Daktari",
    "severity": "replace",
    "bias_type": "occupational_stereotype"
  }]
}

What the API returns

  • check_circleCorrected rewrite — gender-neutral text ready to use
  • check_circlehas_bias_detected — boolean flag for downstream systems
  • check_circleedits array — every change with from/to/severity/bias_type
  • check_circlereason — plain-language explanation for each edit
  • check_circleconfidence — 0.85 if edits made, 0.95 if clean
  • check_circlesource — "rules" | "ml" for audit transparency
  • check_circleskipped_context — terms gated by context checker
