# UI/UX ideas for investor-ready demos

Quick wins that help an investor see impact and write a check.

## Done in this pass

- **Sidebar logo** — JuaKazi branding in the analyse view.
- **Glassmorphism cards** — Frosted-glass style on analyse and result cards (credibility / polish).
- **Impact strip on landing** — "64K+ sentences · 4 languages · AIBRIDGE Gold track" so scale and standards are visible in one line.

## Suggested next (pick what fits)

1. **One-click demo on landing**  
   Add a single "See it in 5 seconds" button that (a) opens `/analyze` in the same tab and (b) auto-fills one strong example (e.g. "The chairman met the waitress") and triggers Analyse, so the first thing they see is a before/after correction. No typing, instant “aha”.

2. **Social proof line**  
   Under the impact strip: "Used by [X] annotators / researchers" or "Powers [partner name]" (when you have permission). One short line is enough.

3. **Clear “Why invest” section**  
   One short block on the landing page: problem (bias in African-language content), solution (detect + correct + explain), traction (64K sentences, AIBRIDGE, languages), and ask (e.g. "Join our round" or "Get in touch"). Keeps the narrative in one place.

4. **Metrics in the product**  
   In the analyse sidebar or footer: "This session: X texts analysed, Y bias instances corrected." Optional: "Total (this browser): …" so repeat visitors see cumulative impact.

5. **Export / share**  
   After analysing, "Download report" (PDF or CSV of original + correction + reason) or "Copy link to this result." Helps investors share the demo with their team.

When you’re satisfied with the UI, squash-merge and continue.
